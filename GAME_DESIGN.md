# GAME DESIGN DOCUMENT

## 1. 게임 개요

그림마피아는 실시간 멀티플레이 그림 추리 게임이다. 플레이어는 시민 또는 마피아 역할을 받고, 자신의 제시어를 직접 보여주지 않은 채 그림으로만 힌트를 남긴다.

현재 실제 게임 설계의 핵심은 다음과 같다.

- 시민은 `행동 + 피사체` 형태의 완성형 제시어를 받는다.
- 마피아는 시민 제시어와 헷갈릴 수 있도록 유사하지만 다른 완성형 제시어를 받는다.
- 현재 턴 플레이어만 그림 입력이 가능하다.
- 다른 플레이어는 그 과정을 실시간으로 본다.
- 각 플레이어는 자신의 턴이 끝나면 해당 라운드의 그림 결과 1장을 남긴다.

## 2. 플레이어 수

- 기본 최소 3명
- 기본 최대 10명
- 테스트 모드에서는 1명 시작 허용

## 3. 역할

- 시민
- 마피아

## 4. 제시어 시스템

### 4-1. 기본 형식

제시어는 항상 `행동 + 피사체` 형식이다.

예시:

- 시민: 먹는 고양이
- 마피아: 마시는 강아지

### 4-2. 지급 규칙

게임 시작 시 `PromptPair` 1개를 선택한다.

- 시민: `citizenAction + citizenSubject`
- 마피아: `mafiaAction + mafiaSubject`

마피아에게 시민 제시어 일부만 따로 공개하는 규칙은 사용하지 않는다.

### 4-3. 데이터 설계 원칙

- `actionPairs`와 `subjectPairs`를 분리한다.
- `allowedCategories`로 자연스러운 조합만 생성한다.
- 카테고리별 데이터 추가만으로 제시어 수를 크게 확장할 수 있어야 한다.
- 단어는 그림으로 표현 가능하고, 초등학생도 이해하기 쉬운 한국어를 우선한다.

## 5. 게임 상태 흐름

실제 상태 값은 아래 다섯 단계다.

`waiting -> playing -> voting -> result -> ended`

- `waiting`: 참가자 모으기, 방장 설정 변경
- `playing`: 턴 기반 그림 입력 진행
- `voting`: 생존 플레이어 투표 (최대 60초, 전원 투표 시 즉시 종료, **채팅 가능**)
- `result`: 탈락 결과와 마피아 정답 입력 처리
- `ended`: 승리 팀 공개, 재시작 가능

## 6. 턴 시스템

- 시작 시 `turnOrder`를 무작위로 생성한다.
- `turnIndex`로 현재 차례를 가리킨다.
- 생존 플레이어가 한 번씩 그림을 그리면 투표 단계로 넘어간다.
- 시간 종료 또는 수동 종료 시 다음 턴으로 진행한다.

## 7. 그림판 규칙

### 7-1. 입력 권한

- 현재 턴 플레이어만 펜, 지우개, 색상, 굵기 입력 가능
- 다른 플레이어는 읽기 전용으로 그림 상태만 본다

### 7-2. 실시간 공유와 턴별 저장

이 프로젝트는 `실시간 공유`와 `턴별 저장`이 동시에 존재한다.

- 현재 턴 플레이어가 stroke를 추가하면 `drawingsByPlayer/{playerId}` 문서에 누적 저장된다.
- 다른 플레이어는 같은 컬렉션을 실시간 구독해 그려지는 과정을 바로 본다.
- 턴 종료 시 해당 플레이어 문서는 그 라운드의 최종 결과 캔버스로 남는다.
- 다음 플레이어는 새 빈 캔버스에서 시작한다.
- 라운드 종료 후에는 플레이어별 그림을 다시 넘겨 보며 비교할 수 있다.

## 8. 타이머

- draw time: 30초 또는 60초, 방장이 대기방에서 선택
- vote time: 최대 60초 (전원 투표 완료 시 즉시 종료)
- mafia guess time: 20초 고정

## 9. 투표 단계

- 모든 생존 플레이어의 그림 턴이 끝나면 `voting`으로 전환
- 생존 플레이어만 1인 1표 행사 가능
- `넘어가기` 선택 가능
- 모든 투표가 완료되면 즉시 집계, 만료(최대 60초) 시에도 집계

## 10. 결과 및 승리 조건

### 10-1. 탈락 없음

다음 조건이면 탈락자가 없다.

- 동률
- `넘어가기` 최다

### 10-2. 마피아 승리

- 마피아 수와 시민 수가 1:1이 되면 즉시 마피아 승리

### 10-3. 시민이 마피아를 탈락시킨 경우

- 마피아에게 시민 제시어를 맞힐 마지막 기회를 준다
- 정답이면 마피아 역전 승리
- 오답 또는 시간 초과면 시민 승리

## 11. 예외 상황

- 대기방 이탈: player document 삭제
- 게임 중 이탈: `alive=false`
- 방장 이탈: 방 종료 처리
- 새로고침: 저장된 플레이어 컨텍스트 기준 복구 시도

## 12. UX 원칙

- 시스템 안내는 모달 또는 인라인 상태 UI로 처리
- 모바일에서도 제시어, 타이머, 현재 턴, 캔버스, 투표 UI가 한 흐름으로 보이도록 구성

## 13. 채팅 가능 상태

채팅 기능은 게임 상태에 따라 제어된다.

| 상태 | 채팅 |
|---|---|
| `waiting` | ✅ 가능 |
| `playing` | ❌ 불가 (그림 그리는 단계) |
| `voting` | ✅ 가능 |
| `result` | ❌ 불가 |
| `ended` | ❌ 불가 |
- 버튼 텍스트는 줄바꿈되지 않도록 유지
