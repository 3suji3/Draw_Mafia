# GAME DESIGN DOCUMENT
## Project: 그림마피아 (Draw Mafia)

---

# 1. 게임 개요

그림마피아는 **멀티플레이 실시간 웹 게임**이다.

플레이어는 그림을 통해 서로의 정체를 추리한다.

- 시민은 같은 제시어를 공유한다.
- 마피아는 시민 제시어와 유사하지만 다른 완성형 제시어를 받는다.
- 그림을 통해 정체를 숨기거나 추리해야 한다.

---

# 2. 플레이어 수

최소: 3명  
최대: 10명

개발 및 테스트를 위해 **1인 테스트 모드 허용**

---

# 3. 역할

플레이어 역할

- 시민 (Citizen)
- 마피아 (Mafia)

---

# 4. 제시어 시스템

제시어는 두 가지 요소로 구성된다.

행동 + 피사체

예시

먹는 고양이
자는 강아지
뛰는 사람

---

### 시민/마피아 지급 규칙

게임 시작 시 PromptPair를 1개 선택한다.

- 시민: citizenAction + citizenSubject
- 마피아: mafiaAction + mafiaSubject

두 제시어는 행동/피사체가 서로 유사하지만 완전히 동일하지 않다.
마피아에게 부분 힌트(행동만 또는 피사체만)는 제공하지 않는다.

---

# 5. 게임 흐름

게임은 다음 단계로 진행된다.

WAITING
↓
START
↓
ROLE_ASSIGN
↓
DRAWING
↓
VOTING
↓
RESULT
↓
NEXT_ROUND 또는 GAME_END

---

# 6. 턴 시스템

게임 시작 시 **랜덤 턴 순서 생성**

각 플레이어는 한 번씩 그림을 그린다.

예

A → B → C → D

모든 생존 플레이어가 그리면 투표 단계로 이동한다.

---

# 7. 그림판 규칙

한 턴에 **한 명만 그림 가능**

그림 기능

- 펜
- 지우개
- 색 선택

모든 플레이어는 그림을 볼 수 있다.
