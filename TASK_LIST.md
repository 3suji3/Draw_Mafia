# TASK LIST
## Project: 그림마피아 (Draw Mafia)

이 문서는 AI가 현재 어떤 작업을 먼저 구현해야 하는지 판단하기 위한 작업 우선순위 문서이다.

AI는 반드시 이 문서의 순서를 기준으로 구현을 진행해야 한다.

한 번에 모든 기능을 구현하지 않는다.

---

# 전체 진행 원칙

- 항상 현재 단계의 목표만 구현한다.
- 다음 단계 기능은 TODO만 남기고 실제 구현은 하지 않는다.
- 파일은 필요한 만큼만 추가한다.
- 각 단계는 실행 가능한 상태여야 한다.
- 기능 구현 전, 기존 구조를 먼저 확인한다.
- copilot-instructions.md, GAME_DESIGN.md, FIREBASE_SCHEMA.md, ARCHITECTURE.md를 반드시 참고한다.

# AI 응답 규칙

AI는 작업 요청을 받으면 다음 순서로 답변해야 한다.

1. 지금 요청이 어느 Phase인지 판단
2. 현재 Phase 범위 안에서만 구현
3. 변경 파일 목록 제시
4. 각 파일 역할 설명
5. 전체 코드 작성
6. 실행 방법 제시
7. 다음 Phase는 구현하지 말고 제목만 제안

# 금지 사항

- 아직 요청하지 않은 다음 Phase를 미리 구현하지 말 것
- 구조를 임의로 바꾸지 말 것
- alert() 사용 금지
- 과도한 라이브러리 추가 금지
- Firebase 구조 임의 변경 금지
